<objective>
Establish the data layer for the v2 Quotation module: extend existing `crm_quotations` and `crm_quotation_line_items` tables with the columns needed for revisions, GST types, insurance, inclusions/exclusions, and payment schedule; create five new tables for `company_settings` and the four DB-driven template types (T&C, Include, Exclude, Payment Schedule); seed default values extracted from the existing `QuotationBuilder.tsx` and `QuotationPDFTemplate.tsx` so the operations team has a working baseline on day one.

Purpose: This is the foundation for every other phase. The user's #1 pain point (hardcoded T&C and Include/Exclude) is solved by the `tc_templates` / `include_templates` / `exclude_templates` tables. The new `company_settings` singleton row feeds the PDF header, the bank block, the logo, and the default validity/GST rates in later phases.

Output: Three new files on disk + database state after running the migration.
</objective>
